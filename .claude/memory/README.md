# memory/

Durable notes *about the project* that are worth re-reading — not transient session memory (that
lives in `~/.claude/projects/.../memory/` and is out of scope here).

- `graphs/` — dependency maps, architecture diagrams.
- `lineage/` — how the architecture got here; why decisions were made.
- `logs/` — curated session/debug summaries kept on purpose.

Add a file when a piece of hard-won context would otherwise be re-derived from scratch. Facts about
*rules* belong in `rules/global/primer-fidelity.md` instead — that one auto-loads.
